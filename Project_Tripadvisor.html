$\Huge{Recommender\ \ system\ \ for\ \ restaurant\ \ choice}$

$\huge{Introduction}$

Where am I going to eat tonight ?

The objective of this project is to get informations on preferences of some customers about different restaurants, and knowing my own opinions about some of these restaurants, to be able to propose a recommendation in order to go and eat at a restaurant that I will really like most surely. To achieve this, I have decided to scrap the website tripadvisor in order to get the ratings of users about restaurants (these ratings go from 1 if they didn't like it to 5 if they loved it), and then to transform these data into a very large and sparse matrix in order to create a recommender system. It means that we will be able throught some machine learning procedures to guess for each user (myself included) the ratings they would have given to restaurants if they went to eat in there, and with this information to propose the top restaurants where they would like to eat, but they don't know it yet.

$\huge{Web scraping}$

In order to build my dataset, I have scraped the 300 first pages of tripadvisor about restaurants in Paris. There are in total more than 500 pages but this gave me enough data and the further you go on those pages, the lesser reviews you get so the last 200 pages are not that usefull for my study. Each page contains 30 restaurants with the links to their own pages, and then when you go to one specific restaurant, you will have at the bottom the first 10 reviews, and the links to the other pages of reviews. Some restaurants have up to 3000 reviews, but most of them only have around 400-500. Needless to say, this procedure took literally for ever (four or five days with two computers), especially because a typo made me do it twice, and also because it seemed like tripadvisor changed some tags during the process, so I had to re-check everything and run it again since it wouldn't work otherwise.

$\Large{\text{Content based methods}}$

In the real world obviously the methods above are not one used to build good recommender systems, since they are really not accurate. Two main methods are used in practice, the collaborative filtering method and the content based method. The collaborative filtering method is the one we are going to focus on from now on, and is based only on interactions between users and products (in our case between users and restaurants). We can distinguish two approaches within that method, the memory based apporach and the model based approach. In order to apply the methods to our data, we will use a python library called "surprise" (for Simple Python Recommendation System Engine), which is specially made to apply machine learning algorithms to large sparse matrix like we have. This library is really easy to use and looks a lot like scikit learn (with methods such as gridSearch, or train_test_split for example). Let's start by loading our data into a surprise object:

$\large{\text{Memory based approach}}$

This approach assumes no latent model and uses only information from the user-item interaction to predict new recommendation. The most famous algorithm for this approach is KNN. It can be computed for user-user similarities, or item-item similarities. For user-user (the one we will focus on), it will compute the distances between all users, where the distance is calculated by default as the mean square difference (msd) of ratings over all items (it takes into account only items for which both users gave ratings, meaning that is there is a NaN value in either of them, the rating difference on this item will not be calculated):

$$d^{msd}(u,v) = \frac{1}{\mid J_{uv}\mid}\sum_{j\in J_{uv}}{(r_{uj} - r_{vj})^2} $$

Where $r_{uj}$ is the rating given by user $u$ to restaurant $j$, $r_{vj}$ is the rating given by user $v$ to restaurant $j$ and $J_{uv}$ is the space of restaurant for which both user $u$ and $v$ gave ratings.

The predicted value of rating for user $i$ to restaurant $j$ will then be the average rating given to that restaurant over the $k$ neighbors of $i$.

By default, the KNN is trained with k=40 (number of neighbors), min_k=1 (minimum of of neighbors to take into account for aggregation, if there are not enough neighbors, the prediction is set to the global mean of all ratings), and the measure of similarities "msd" (mean square difference).

We will do a grid search in order to find the best parameters of the KNN method to fit our data: we will do some cross validation with different values of k, min_k and with two measures of similarities: the "msd" measure and the "cosine" measure, which is defined as:

$$d^{cosine}(u,v) = \frac{\sum\limits_{j\in J_{uv}}{r_{uj}.r_{vj}}}{\sqrt{\sum\limits_{j\in J_{uv}}{r_{uj}^2}}\sqrt{\sum\limits_{j\in J_{uv}}{r_{vj}^2}}} $$

This is probably due to the fact that most of the real ratings are concentrated around the mean (around 4.0) and thus giving the average rating all the time ensure that we are not very far from each actual ratings (especially because the 1.0 or 2.0 ratings are quit rare, which is not that significant when computing the mean square error). Unfortunatly the surprise library doesn't allow us to be free on this part and only propose four different measures of error: the mse (mean square error), the rmse, the mae (mean absolute error) and the fcp (fraction of concordant pairs).

$\large{\text{Model based approach}}$

Model based apporaches, like matrix factorization, are still based on user-item interactions but assume a latent model is supposed to explain these interactions. In matrix factorization, the idea is to find two small-dimensional and dense matrices to represent both users and items in order to reconstruct the initial matrix by computing the dot product of the two.

In other words, we search $A\in M_{n,k}(\mathbb{R})$ and $B\in M_{k,p}(\mathbb{R})$ such that $M = A.B + \epsilon$ where M is our matrix rating. $k$ represents a property in restaurant, for example the fact that it is cheap, and if a restaurant $j$ has that property the value of $b_{k,j}$ will be high. $k$ represents also how much the user values that feature. So if a user $i$ likes going to cheap restaurant, $a_{ik}$ will be high.

So we search: $$(A,B) = \underset{B\in M_{k,p}(\mathbb{R})}{\underset{A\in M_{n,k}(\mathbb{R})}{\mathrm{argmin}}}\sum_{(i,j)\in E}{\left(M_{i,j} - A_iB_j\right)^2} $$

Where $E = \{(i,j)\in\mathbb{R}^{n\times p}$ such that $M_{i,j}\neq nan\}$

($E$ is the space of index where $M$ is full)

Since the vectors of $A$ and $B$ can take any value, and in order to avoid overfitting, we need to add regularization to our loss function, with a penalty coefficient $\lambda$ on the norm $\|A_i\|^2$ and $\|B_j\|^2$.

Hence we want to minimize the expression: $$\underset{A,B}{\mathrm{min}}\ \ \frac{1}{2}\sum_{(i,j)\in E}{\left(M_{i,j} - A_iB_j\right)^2} + \frac{1}{2}\lambda\left(\|A_i\|^2 + \|B_j\|^2\right)$$

But what is done in the surprise library is a bit more subtle than that, because it takes something important that wasn't mentioned so far: the bias.

In fact, the two matrices we want to find here are supposed to explain how the interactions between users and restaurants work and how they lead to produce the rating values. However, much of the real variation in rating values is due to particular effects amongst users or restaurant, and are independant of these variations. These effects are called biases. For example, some users tend to give higher rates than other for the same level of satisfaction, which will result in differences of ratings which are not significant regarding our models and objectives. In order to take this effect into account, we model the bias by approaching $M_{ij}$ by $A_i.B_j + \mu + \beta_i + \beta_j$, where $\mu$ is the overall rating average (it can be computed directly from the data), $\beta_i$ is the bias of user $i$ and $\beta_j$ is the bias of restaurant $j$.

The expression we want to minimize becomes now: $$\underset{A,B,\beta}{\mathrm{min}}\ \ \frac{1}{2}\sum_{(i,j)\in E}{\left(M_{i,j} - A_iB_j\right)^2} + \frac{1}{2}\lambda\left(\|A_i\|^2 + \|B_j\|^2\right)$$ with $A\in M_{n,k}(\mathbb{R})$, $B\in M_{k,p}(\mathbb{R})$ and $\beta\in\mathbb{R}^{n\times p}$

It can be done simply with a stochastic gradient descent (we are not going to write the details but the calculation of the gradient is straightforward). This is what is done with the library surprise and the method SVD, which we are going to apply now.
